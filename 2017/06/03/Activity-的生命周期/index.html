Activity 的生命周期

Android 中的四大组件之一， Activity 。

Activity 是 Android 中的四大组件之一，出现的频率也是相当高的，主要是负责与用户交互，理解 Activity 的生命周期有助于更好的开发。

Activity 的状态

Active/Running
这个状态是拥有与用户进行交互能力的、
Paused
当 Activity 失去焦点时，或被部分覆盖，或被不透明 Activity 覆盖，会失去与用户之间的交互能力，但状态信息和成员变量都还在，内存紧张时有被回收的可能。
Stopped
被新的 Activity 覆盖，但状态信息和成员变量都还保存着，其任务栈还存在，还有被重新唤醒的可能。
Killed
被系统回收，任务栈不存在。
